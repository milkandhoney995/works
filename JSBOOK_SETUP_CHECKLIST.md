## 💡 主要な実装特性

1. **動的ルーティング**: `[topic]` フォルダで自動ルート生成
2. **API 統合**: 動的 API ルート `[topic]` で Markdown 配信
3. **クライアント側処理**: fetch API で API から Markdown を取得
4. **コンポーネント再利用**: Layout コンポーネントで共通レイアウト
5. **SCSS モジュール**: Scoped スタイリング
6. **TypeScript**: 型安全なコード

## 📞 トラブルシューティング

### API 404 エラー
**原因**: `_content/` フォルダに Markdown ファイルがない
**解決**: ファイルが存在するか確認し、名前を確認

### スタイルが反映されない
**原因**: SCSS モジュールのインポート問題
**解決**: ファイルパスを確認、キャッシュをクリア

### Markdown が表示されない
**原因**: パーサーが対応していない Markdown 構文
**解決**: `markdownParser.ts` で対応を追加
