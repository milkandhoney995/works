# JSBook - Content Files

このディレクトリは、JSBook で使用されるすべての Markdown コンテンツファイルを格納しています。

## ファイル一覧

### basics.md
JavaScriptの基本文法について、初心者向けの包括的なガイドです。

**カバーしている内容:**
- 変数宣言（var, let, const）
- データ型（プリミティブ型、オブジェクト型）
- 演算子（算術、比較、論理）
- 制御構文（if, switch, ループ）
- 関数（関数宣言、関数式、アロー関数）
- スコープとクロージャ
- テンプレートリテラル

### dom.md
HTML DOM の操作について詳しくカバーしている、このコースの重要なパートです。

**カバーしている内容:**
- DOMの基本概念と構造
- 要素の取得（getElementById, querySelector など）
- 要素の内容変更
- 属性操作
- クラス操作
- スタイル操作
- 要素の作成と追加
- 要素の削除
- 要素の関係性
- パフォーマンス最適化

### events.md
イベント駆動プログラミングに関するすべてを扱っています。

**カバーしている内容:**
- イベントの種類（マウス、キーボード、フォーム、ウィンドウ）
- イベントリスナーの追加
- イベントオブジェクト
- イベント制御（preventDefault, stopPropagation）
- イベントリスナーの削除
- イベント委譲
- キーボードショートカット
- デバウンスとスロットリング

### arrays.md
配列とその強力なメソッドについての詳細ガイドです。

**カバーしている内容:**
- 配列の基本操作
- 配列を変更するメソッド（push, pop, splice など）
- 配列を変更しないメソッド（slice, concat など）
- 反復処理メソッド（forEach, map, filter, reduce）
- 検索メソッド（find, findIndex, some, every）
- 配列の展開（flat, flatMap）
- 実践的なデータ処理例

### advanced.md
JavaScriptの高度なトピックをカバーしています。

**カバーしている内容:**
- 非同期処理（コールバック、Promise、async/await）
- オブジェクト指向（クラス、継承）
- 関数型プログラミング（compose, map, filter, reduce）
- デストラクチャリング
- スプレッド演算子とレスト演算子
- クロージャとモジュールパターン
- this バインディング
- プロトタイプ
- ジェネレータ

## Markdown フォーマット

すべてのファイルは標準的な Markdown フォーマットで記述されています：

```markdown
# H1 見出し
## H2 見出し
### H3 見出し

**太字テキスト**
*斜体テキスト*

`インラインコード`

\`\`\`javascript
// コードブロック
const x = 10;
\`\`\`

- リスト項目
- リスト項目

1. 順序付きリスト
2. 順序付きリスト

| テーブル | ヘッダ |
|--------|--------|
| セル   | セル   |

[リンク](https://example.com)
```

## 新しいコンテンツを追加する方法

1. 新しい Markdown ファイルを作成：`src/app/jsbook/_content/your-topic.md`
2. `src/app/jsbook/_data/topics.ts` にエントリを追加
3. Markdown コンテンツを記述
4. ページが自動的に生成される

## コンテンツガイドライン

### 構造
- H1 は 1 つだけ（ファイルのタイトル）
- H2 で主要なセクションを分割
- H3 でサブセクションを分割

### コード例
- 実行可能なコード例を含める
- 説明的なコメントを追加
- エラーケースも示す

### 長さ
- セクションは 200-500 語が目安
- スクロール量を考慮
- 複数の小さなセクションの方が一つの大きなセクションより良い

### スタイル
- 明確で簡潔な言語を使用
- `we` や `our` ではなく `you` を使用
- 技術用語は最初の使用時に説明

## コンテンツの保守

- 定期的にコンテンツを確認し、最新のベストプラクティスに合わせる
- 古い API や方法は注記を付ける
- 実装例が正確であることを確認
- リンク切れを定期的にチェック

## SEO と メタデータ

- ファイル名は検索に応答的である必要があります
- 見出しは意味のあるキーワードを含む
- 最初の段落は内容の要約

## ライセンスと帰属

外部のリソースから引用する場合は、適切に帰属を表示してください。
